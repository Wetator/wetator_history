Wenn die Anzahl der fehlgeschlagenen Wetator-Tests größer als dieser Grenzwert ist, wird der Build als fehlgeschlagen markiert. Beispiel: Der Wert 0 bedeutet, dass der Status des Builds verändert wird, wenn mindestens ein Wetator-Test fehlschlägt. Wird das Feld leer gelassen, wird der Status des Builds niemals aufgrund der Anzahl der fehlgeschlagenen Wetator-Tests verändert. (Standard: niemals)
